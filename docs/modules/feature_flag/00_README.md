# Onboarding Guide: Feature Flag Management System

Welcome to the Awo ERP Feature Flag Management System!

This guide is designed for new developers, SREs, and technical writers to quickly get up to speed on how our feature flag system works, how to use it effectively, and how to manage it.

## Recommended Reading Order

Please follow the documents in numerical order for a structured learning path.

1.  **[Introduction](./onboarding/01_introduction.md)**
    *   **What you'll learn:** A high-level overview of the feature flag module, its purpose, and the business value it provides.

2.  **[02_core_concepts_and_architecture](./onboarding/02_core_concepts_and_architecture.md)**
    *   **What you'll learn:** A breakdown of the key concepts like flags, rollouts, and targeting. You'll also get an inside look at the system's architecture and data flow.

3.  **[03_developer_quickstart](./onboarding/03_developer_quickstart.md)**
    *   **What you'll learn:** A practical guide to creating and evaluating flags within the application code. This is essential for all developers.

4.  **[04_admin_api_manual](./onboarding/04_admin_api_manual.md)**
    *   **What you'll learn:** A detailed manual for the administrative API, covering bulk operations, health checks, and emergency controls. Essential for SREs and system administrators.

5.  **[05_lifecycle_gotchas_and_improvements](./onboarding/05_lifecycle_gotchas_and_improvements.md)**
    *   **What you'll learn:** Best practices for managing a flag's lifecycle, common pitfalls to avoid, and potential areas for future system improvements.
